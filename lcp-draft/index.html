LIQUID CONTEXT PROTOCOL

In-process tool execution for local AI agents.

RFC — Status: Seeking Contributors

01. THE PROBLEM

// Every MCP tool call looks like this:

LLM generates text
→ serialize to JSON
→ HTTP round-trip to external process
→ wait
→ deserialize response
→ inject result text into context
→ resume generation

For local models making 20–50 tool calls per task, this bottleneck compounds. The network overhead can exceed the inference time itself.

02. THE SOLUTION

Move tool execution into the inference process.

LCP tools are .wasm binaries loaded directly into the inference engine's address space at startup (e.g., llama.cpp with an embedded Wasmtime runtime).

// LCP tool call pipeline:

LLM emits <|lcp_call|> token

→ inference loop intercepts (no network)

→ WASM module executes in-process (~microseconds)

→ result injected into context buffer

→ generation resumes

03. ARCHITECTURE

In-Process Execution

No HTTP. No JSON-RPC. Tools run as WASM modules inside the inference engine's memory space. The result is available before the next token is sampled.

Near-Zero Latency

Tool call overhead drops from network-bound (10–500ms) to in-process (~microseconds). For autonomous agents, this is a fundamental throughput improvement.

WASM Sandbox

WebAssembly provides memory isolation by design. Tools cannot access engine memory, the filesystem, or the network unless explicitly granted via WASI capabilities.

Offline Capable

Designed for open-weight models running locally. No external API calls required for tool use. Works on air-gapped machines and edge hardware.
